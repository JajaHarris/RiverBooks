# Plans

- Plans are created in `.agents/plans`
- Plans are created in numbered phases
- Phases and smaller tasks have open checkboxes showing incomplete status initially
  - Agents can mark tasks as completed or failed/canceled when implementing the plan
- Plans are created in markdown format
- Plan files are named following the convention: `YYYY-MM-DD-short-description.md`
- Fully completed plans are moved to `.agents/plans/completed`
- Valid markdown uses newlines before and after headings, lists, and code fences
